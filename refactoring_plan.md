# 마늘봇 리팩터링 계획

## 현재 상태 분석

### 문제점

- **main.py**: 9,583줄의 거대한 단일 파일
- **일관성 없는 구조**: 명령어가 main.py와 commands/ 폴더에 혼재
- **설정 관리 부재**: 하드코딩된 설정값들이 산재
- **모듈 간 의존성**: define.py에 모든 것이 집중되어 순환 참조 위험

## 목표

- **모듈화**: 관심사별로 명확한 분리
- **일관성**: 통일된 명령어 및 이벤트 핸들러 구조
- **유지보수성**: 설정 중앙화 및 의존성 정리
- **확장성**: 새로운 기능 추가 시 명확한 구조

## 단계별 실행 계획

### Phase 1: 기반 구조 설정 (1주)

1. **설정 중앙화**

   - config/ 모듈 생성
   - 하드코딩된 설정값들을 settings.py로 이동
   - 환경변수 관리 체계화

2. **핵심 모듈 구성**
   - core/ 모듈 생성
   - Bot 인스턴스 분리
   - 데이터베이스 연결 모듈화

### Phase 2: 이벤트 핸들러 분리 (1주)

1. **이벤트 핸들러 모듈화**
   - events/ 모듈 생성
   - main.py의 @bot.event 함수들을 카테고리별로 분리
   - 이벤트 자동 등록 시스템 구축

### Phase 3: 명령어 체계 통일 (2주)

1. **명령어 기본 구조 설계**

   - 명령어 기본 클래스 생성
   - 공통 검증 로직 추상화
   - 에러 처리 통일

2. **명령어 카테고리별 분리**
   - 기존 commands/ 폴더 재구성
   - main.py의 명령어들을 카테고리별로 이동
   - 명령어 자동 등록 시스템

### Phase 4: 서비스 레이어 구축 (1주)

1. **비즈니스 로직 분리**
   - services/ 모듈 생성
   - 복잡한 비즈니스 로직을 서비스로 분리
   - 재사용 가능한 서비스 인터페이스 구축

### Phase 5: UI 컴포넌트 정리 (1주)

1. **UI 컴포넌트 모듈화**
   - discord.ui 관련 컴포넌트 분리
   - 재사용 가능한 UI 패턴 구축

### Phase 6: 유틸리티 및 마무리 (1주)

1. **공통 유틸리티 정리**
   - 중복 코드 제거
   - 헬퍼 함수 정리
   - 테스트 코드 작성
